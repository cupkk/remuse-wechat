## 🔧 环境要求

### 服务器配置
| 配置项 | 最低配置 | 推荐配置 | 说明 |
|--------|----------|----------|------|
| CPU | 2核 | 4核以上 | |
| 内存 | 4GB | 8GB以上 | |
| 带宽 | 5M | 10M以上 | 图片和视频上传需要更高带宽 |
| 磁盘 | 100GB | 200GB以上SSD | 用于存储图片、PDF等静态资源 |

### 软件环境
| 软件 | 版本要求 | 说明 |
|--------|----------|------|
| Node.js | >= 18.x | 推荐LTS版本 |
| npm | >= 9.x | 随Node.js安装 |
| Nginx | >= 1.20 | 反向代理和静态文件服务 |
| Redis | >= 6.x | 缓存和会话存储 |
| SQLite3 | >= 3.37 | 数据库 |
| PM2 | >= 5.x | Node.js进程管理 |
| Docker（可选） | >= 20.x | 容器化部署可选 |

## 🚀 部署步骤

### 1. 基础环境准备
```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装基础依赖
sudo apt install -y build-essential git curl wget nginx redis-server sqlite3

# 安装Node.js 18 LTS
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node -v  # v18.x
npm -v   # 9.x
redis-server -v  # 6.x
nginx -v  # 1.20+

# 安装PM2
sudo npm install -g pm2
pm2 startup
```

### 2. 项目下载与配置
```bash
# 创建项目目录
sudo mkdir -p /var/www/remuse
sudo chown $USER:$USER /var/www/remuse
cd /var/www/remuse

# 克隆项目代码
git clone <你的项目仓库地址> .

# 安装依赖
cd backend
npm install --production

cd ../fronted
npm install --production

cd ../remuse-miniprogram
npm install --production
```

### 3. 环境变量配置
#### 后端配置 (`backend/.env`)
```env
# 基础配置
NODE_ENV=production
PORT=3001
BASE_URL=https://你的域名
JWT_SECRET=你的JWT密钥（随机字符串）
JWT_EXPIRES_IN=7d

# 微信小程序配置
WECHAT_APPID=你的小程序AppID
WECHAT_SECRET=你的小程序AppSecret

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=你的Redis密码（如有）
REDIS_DB=0

# 数据库配置
DB_PATH=../data/production.sqlite

# AI服务配置
GEMINI_API_KEY=你的Gemini API密钥
AI_REQUEST_TIMEOUT=30000

# 存储配置
UPLOAD_PATH=/var/www/remuse/uploads
MAX_UPLOAD_SIZE=10MB
```

#### 前端配置 (`fronted/.env.production`)
```env
VITE_API_BASE_URL=https://你的域名/api
VITE_APP_TITLE=再生博物馆
```

### 4. 项目构建
```bash
# 构建前端
cd /var/www/remuse/fronted
npm run build

# 构建后端
cd /var/www/remuse/backend
npm run build
```

### 5. 静态资源目录配置
```bash
# 创建上传目录
sudo mkdir -p /var/www/remuse/uploads
sudo mkdir -p /var/www/remuse/data
sudo chown -R www-data:www-data /var/www/remuse/uploads
sudo chown -R www-data:www-data /var/www/remuse/data
sudo chmod -R 755 /var/www/remuse/uploads
```

### 6. PM2服务配置
#### 后端服务配置 (`/var/www/remuse/pm2.config.js`)
```javascript
module.exports = {
  apps: [
    {
      name: 'remuse-backend',
      script: 'dist/server.js',
      cwd: '/var/www/remuse/backend',
      instances: 'max',
      exec_mode: 'cluster',
      env: {
        NODE_ENV: 'production',
        PORT: 3001
      },
      error_file: '/var/log/remuse/backend-error.log',
      out_file: '/var/log/remuse/backend-out.log',
      log_date_format: 'YYYY-MM-DD HH:mm:ss',
      merge_logs: true,
      autorestart: true,
      watch: false,
      max_memory_restart: '1G'
    }
  ]
}
```

```bash
# 创建日志目录
sudo mkdir -p /var/log/remuse
sudo chown $USER:$USER /var/log/remuse

# 启动后端服务
cd /var/www/remuse
pm2 start pm2.config.js

# 保存PM2配置
pm2 save
```

### 7. Nginx配置
#### 站点配置 (`/etc/nginx/sites-available/remuse`)
```nginx
server {
    listen 80;
    server_name 你的域名;

    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name 你的域名;

    # SSL证书配置（使用Let's Encrypt）
    ssl_certificate /etc/letsencrypt/live/你的域名/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/你的域名/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 安全头
    add_header X-Frame-Options DENY;
    add_header X-Content-Type-Options nosniff;
    add_header X-XSS-Protection "1; mode=block";
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self'";
    add_header Referrer-Policy strict-origin-when-cross-origin;

    # 前端静态文件
    location / {
        root /var/www/remuse/fronted/dist;
        try_files $uri $uri/ /index.html;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API请求代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
        client_max_body_size 10M;
    }

    # 静态资源
    location /uploads/ {
        alias /var/www/remuse/uploads/;
        expires 1y;
        add_header Cache-Control "public, immutable";
        add_header Access-Control-Allow-Origin *;
    }

    # 监控端点保护（只允许本地访问）
    location /metrics {
        allow 127.0.0.1;
        deny all;
        proxy_pass http://127.0.0.1:3001/metrics;
    }

    # 日志
    access_log /var/log/nginx/remuse-access.log;
    error_log /var/log/nginx/remuse-error.log;

    # 错误页面
    error_page 404 /index.html;
    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /usr/share/nginx/html;
    }
}
```

#### 启用配置
```bash
# 启用站点
sudo ln -s /etc/nginx/sites-available/remuse /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

### 8. SSL证书配置（Let's Encrypt）

## 🔄 小程序部署
### 1. 小程序配置
1. 登录微信公众平台小程序后台
2. 配置服务器域名：
   - request合法域名：`https://你的域名`
   - uploadFile合法域名：`https://你的域名`
   - downloadFile合法域名：`https://你的域名`
3. 配置业务域名（如果需要web-view）
4. 配置服务器域名白名单

### 2. 代码提交审核
1. 在微信开发者工具中打开 `remuse-miniprogram` 目录
2. 修改 `services/api.js` 中的 `baseURL` 为正式域名：
   ```javascript
   const baseURL = 'https://你的域名/api'
   ```
3. 点击上传，填写版本号和更新说明
4. 在小程序后台提交版本审核
5. 审核通过后发布上线

## 🛠️ 运维指南

### 常用命令
```bash
# 查看服务状态
pm2 status
pm2 logs remuse-backend
pm2 monit

# 重启服务
pm2 reload remuse-backend

# Nginx相关
sudo nginx -t
sudo systemctl reload nginx
sudo systemctl restart nginx

# 查看日志
tail -f /var/log/remuse/backend-error.log
tail -f /var/log/nginx/remuse-access.log

# 数据库备份
sqlite3 /var/www/remuse/data/production.sqlite ".backup /var/backup/remuse-$(date +%Y%m%d).sqlite"

# Redis备份
redis-cli BGSAVE
cp /var/lib/redis/dump.rdb /var/backup/redis-$(date +%Y%m%d).rdb
```

### 定期任务配置 (`crontab -e`)
```bash
# 每日凌晨2点数据库备份
0 2 * * * sqlite3 /var/www/remuse/data/production.sqlite ".backup /var/backup/remuse-$(date +\%Y\%m\%d).sqlite"

# 每日凌晨2点30分Redis备份
30 2 * * * redis-cli BGSAVE && cp /var/lib/redis/dump.rdb /var/backup/redis-$(date +\%Y\%m\%d).rdb

# 每周一凌晨3点清理7天前的备份
0 3 * * 1 find /var/backup -name "*.sqlite" -mtime +7 -delete
0 3 * * 1 find /var/backup -name "*.rdb" -mtime +7 -delete

# 每日凌晨4点清理临时文件
0 4 * * * find /var/www/remuse/uploads/tmp -type f -mtime +1 -delete
```

### 故障排查
#### 服务无法启动
1. 检查PM2日志：`pm2 logs remuse-backend`
2. 检查环境变量配置是否正确
3. 检查Redis服务状态：`systemctl status redis-server`
4. 检查端口是否被占用：`netstat -tlnp | grep 3001`

#### API请求502错误
1. 检查后端服务是否正常：`pm2 status`
2. 检查Nginx配置：`nginx -t`
3. 查看Nginx错误日志：`tail -f /var/log/nginx/remuse-error.log`

#### 图片无法上传
1. 检查上传目录权限：`ls -la /var/www/remuse/uploads`
2. 检查目录所有者是否为www-data
3. 检查Nginx配置中的`client_max_body_size`设置
4. 检查后端配置中的`MAX_UPLOAD_SIZE`

## 🚨 紧急回滚方案
### 代码版本回滚
```bash
# 进入项目目录
cd /var/www/remuse

# 查看提交历史
git log --oneline

# 回滚到指定版本
git reset --hard <commit-hash>

# 重新构建部署
cd fronted && npm run build
cd ../backend && npm run build && pm2 reload remuse-backend
```

### 数据恢复
```bash
# 恢复SQLite数据库
sqlite3 /var/www/remuse/data/production.sqlite ".restore /var/backup/remuse-xxxxxx.sqlite"

# 恢复Redis数据
systemctl stop redis-server
cp /var/backup/redis-xxxxxx.rdb /var/lib/redis/dump.rdb
systemctl start redis-server
```
