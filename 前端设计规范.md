# Remuse 再生博物馆小程序前端界面设计说明文档 v0.1

## 一、整体设计定位

Remuse 小程序不应该做成 Web 端的缩小版，也不应该做成一个功能入口堆叠的工具箱。新的产品逻辑已经从“用户选择再生工坊功能”调整为“用户上传物品、讲述故事，系统自动判断最适合的处理方式并给出情感化结果”。也就是说，小程序的核心体验应该是 **被理解感**，而不是 **功能选择感**。新的 PRD 明确强调，产品以“物品故事”为核心输入，以“情感再生”为核心输出，用户上传物品后先讲故事，再由系统结合图像和故事自动判断处理方式。

因此，小程序前端的整体气质应该是：

> 像一个随身的夜间数字博物馆，用户每次打开都可以把一件旧物交给它，让它帮用户识别、倾听、判断、保存和再生。

界面不宜做成传统相册、不宜做成普通社区，也不宜做成 SaaS 后台。它应该延续 Web 端的深色底、荧光强调色、博物馆感、终端感和数字档案感，但所有页面都要移动端化、单列化、轻操作化。此前小程序 PRD 也已经明确，小程序不应全量搬 Web，而要保留最适合移动端的核心体验，并且页面要适配微信小程序触控和单列结构。

本版小程序的主结构建议改为：

| 底部 Tab | 页面定位                    |
| ------ | ----------------------- |
| 首页     | 每日盲盒、上传入口、最近记忆、灵感入口     |
| 上传     | 拍照 / 选图 / 语音讲故事 / AI 分析 |
| 灵感     | 真实社区：看帖子、发帖子、一键同款、交换    |
| 我的     | 我的藏品、故事、成就、盲盒、交换、会员     |

我不建议继续把“再生工坊”作为一级页面。再生工坊应该退到系统能力层，用户不需要自己判断“我要做贴纸、卡片还是拼豆”，而是由系统在结果页直接给出主推荐和备选推荐。新的 PRD 也明确要求，处理路径应由物品图像、故事内容、情绪强度、稀有度和价值共同决定，输出主推荐、备选推荐、推荐理由和下一步按钮。

## 二、核心用户路径与页面关系

小程序的核心路径应该非常短，用户从打开到完成一次“物品转生”最好控制在 4 到 6 个关键动作内。

主路径是：

> 打开首页 → 点击上传 → 拍照或选图 → AI 初识别 → 语音讲故事 → AI 分析故事和情绪 → 结果页给出推荐方案 → 保存 / 生成 / 发布到灵感广场。

这里最重要的是“先故事，后处理”。新的 PRD 明确写到，核心闭环是“拍照上传 → 语音讲故事 → AI 分析物品 + 故事 + 情绪 → 输出处理建议 → 用户执行下一步”，并且理念是物品本身不是重点，重点是物品背后发生过什么。

所以页面关系应该这样设计：

| 页面    | 进入方式                     | 完成后的去向                 |
| ----- | ------------------------ | ---------------------- |
| 首页    | 打开小程序默认进入                | 上传页、灵感广场、今日盲盒详情        |
| 上传故事页 | 首页 CTA / 底部上传 Tab / 一键同款 | 结果页                    |
| 结果页   | 上传分析完成后进入                | 保存到我的、生成成果、发布到灵感、发起交换  |
| 灵感广场  | 底部 Tab 进入                | 帖子详情、发帖页、一键同款上传        |
| 帖子详情页 | 点击灵感流卡片进入                | 一键同款、收藏、交换、评论          |
| 发帖页   | 结果页发布 / 灵感广场发布按钮         | 灵感广场详情页                |
| 我的页   | 底部 Tab 进入                | 我的藏品、我的故事、我的成就、我的交换、会员 |

这一版最关键的变化是：**展馆不一定要作为底部一级 Tab**。因为现在社区和上传是增长重点，展馆可以收进“我的”页面，命名为“我的藏品”或“我的记忆馆”。这样底部结构更轻，也更符合新 PRD 中“首页、上传后故事页、结果页、灵感广场、我的页面”的页面结构。

## 三、页面设计说明

### 1. 首页：情绪入口 + 每日回访 + 上传分发

首页不要做成普通功能宫格。它应该像用户进入 Remuse 的“今日展厅”。

页面顶部是品牌区，背景使用深色渐变或暗色展厅感纹理，文案可以是：

> 今天想让哪件旧物重获新生？

下面放一个非常明显的主按钮：

> 拍下我的旧物

这个按钮应该是全页面最强的 CTA，使用荧光黄绿色或青绿色高亮。按钮下方可以有一句柔和提示：

> 说一句它的故事，我们会认真记住。

首页中间放“今日物品盲盒”。这不是普通广告位，而是提高每日打开频次的核心模块。新 PRD 明确提出，每天首页自动推送一个“今日物品盲盒”，从用户过去上传过的藏品中随机抽取一件，生成情感化回顾内容，用于提高每日打开率。

盲盒卡片可以这样设计：

```text
今日物品盲盒
去年的这杯奶茶很甜，
但今天也记得给自己买杯热的。

[打开今日记忆]
```

如果用户没有上传过物品，盲盒模块显示空态：

```text
你的第一件旧物还没有被保存。
从一张票根、一只玩偶、一个包装袋开始吧。
[拍下第一件旧物]
```

首页下半部分放两个轻入口：一个是“最近转生”，展示最近生成的贴纸、卡片或故事；另一个是“灵感广场精选”，用 2 到 4 张社区作品卡片引导用户进入灵感广场。首页不建议放太多功能入口，比如拼豆、改造、表情包、会员、商城全部堆上来，这样会稀释主线。

---

### 2. 上传故事页：拍照后立刻进入“被倾听”的状态

上传页是整个小程序最重要的页面。它不能只是“上传图片 + 表单”，而应该表现为一个连续的 AI 陪伴流程。

页面分为三个状态。

第一个状态是未上传状态。页面中心放一个大面积上传区域，像一个“扫描台”或“展品采集框”。按钮文案建议：

```text
拍照上传
从相册选择
```

旁边可以有说明：

```text
票根、包装、玩偶、旧卡片、徽章都可以。
它不一定贵重，只要你想记住。
```

第二个状态是图片上传后。图片放在页面上方，下面立即显示第一层识别反馈。新的 PRD 要求上传后立即进入识别状态，并在 3 秒内给出第一层反馈，包括识别物品名称、初步分类和温柔提示语。

示例界面：

```text
识别为：奶茶袋
它看起来很适合被认真记住。

愿意和我说说它的故事吗？
```

第三个状态是语音故事输入。语音按钮应该是这个页面的视觉中心，采用按住说话的交互。PRD 要求支持 5 到 30 秒短语音，自动转文字显示，并允许用户手动编辑文本。

界面建议：

```text
用十秒说说它的来历吧

[按住说话]

它属于谁？什么时候得到的？
如果它会说话，它会记得什么？
你也可以只说一句。
```

语音转文字后，不要跳走，而是在原页面显示一张“故事草稿卡”：

```text
我听到的故事
这是去年夏天和朋友一起买的奶茶，当时我们刚考完试……

[修改文字] [继续分析]
```

如果用户说得太短，可以最多追问一次：

```text
你最想记住它的哪一点？
```

但不能强迫用户补充。新 PRD 也明确，如果用户不回答，也允许进入后续分析流程。

---

### 3. 结果页：不要像报告，要像“物品的第二人生说明书”

结果页不是冷冰冰的识别结果页，而应该是整个产品情绪价值最强的页面。它应该告诉用户三件事：

第一，系统理解了这是什么物品。
第二，系统理解了它为什么重要。
第三，系统建议它以什么方式继续存在。

页面顶部应该展示一张视觉效果图，可以是物品原图加轻度设计化处理，也可以是系统生成的卡片封面。标题区域不要写“分析完成”，而可以写：

```text
它最适合被这样继续存在
```

页面主体建议分成四张卡片：

| 模块   | 内容                  |
| ---- | ------------------- |
| 物品识别 | 名称、分类、材质、风格、可能品牌    |
| 故事摘要 | 从用户语音中提炼出来的一段简短故事   |
| 情绪回应 | 一句温柔反馈，不要太 AI 腔     |
| 推荐方案 | 主推荐、备选推荐、推荐理由、下一步按钮 |

示例：

```text
故事摘要
这只空香水瓶来自你第一次独自旅行。它本身已经用完了，但你记住的是那段“终于可以自己决定路线”的自由感。

情绪回应
它不像一个普通瓶子，更像是那段旅程留下的坐标。

推荐方案
主推荐：数字化再生 · 纪念卡片
推荐理由：它的情绪强度较高，且外观具有较强审美特征，适合被做成一张可保存、可分享的记忆卡。
[生成纪念卡片]
```

如果是视觉特征强的物品，比如奶茶袋、包装盒、香水瓶，主推荐可以是创意再生或贴纸。如果是票根、门票，主推荐可以是纪念卡和时间线。如果是玩偶、手办，主推荐可以是收藏展示或表情包。如果是低情感闲置物，可以推荐交换 / 售卖 / 基础纪念卡。新 PRD 对这些生成逻辑已经给出了方向。

结果页底部放三个主要操作：

```text
[保存到我的记忆馆]
[生成推荐成果]
[发布到灵感广场]
```

这里不要再让用户进入“再生工坊”选择一堆功能，而是根据推荐结果直接显示下一步。例如系统推荐贴纸，就显示“生成专属贴纸”；推荐交换，就显示“挂到交换墙”。

---

### 4. 灵感广场：做成真正社区，而不是静态瀑布流

灵感广场是这一版小程序最需要认真做的页面。它不是“发现页”，而是一个围绕旧物转生成果的视觉社区。

页面顶部可以写：

```text
灵感广场
看看别人如何让旧物重获新生
```

下面是分类切换：

```text
推荐 / 贴纸 / 卡片 / 拼豆 / 改造 / 交换
```

主体采用双列瀑布流。新 PRD 明确要求灵感广场是纯视觉驱动的 AIGC 审美灵感流，展示贴纸、拼豆图纸、卡片、再生作品、数字手账页，并采用双列瀑布流、强视觉展示、弱文字说明。

卡片设计建议：

```text
[作品图]
标签：贴纸
一张票根做成了春天感贴纸
@旧物收藏家  128 喜欢
```

每张卡片必须可以点击进入详情页。现在如果只是展示假数据，用户无法点击查看、无法点赞收藏、无法发帖，就不像真实社区。

帖子详情页建议这样设计：

```text
大图 / 多图轮播

一张票根做成了春天感贴纸
来自：演唱会票根
作者：@旧物收藏家

这张票根是去年春天去看演唱会留下的，后来被做成了带花边的记忆贴纸。

[点赞] [收藏]
[一键同款再生]
[我想交换]
```

其中“一键同款再生”应该是详情页最重要的按钮。PRD 中写到，用户看到喜欢的作品后，可以点击“一键同款再生”，系统调起相机、复用当前作品的再生风格，引导用户上传自己的物品。

发帖页不要做成普通“发动态”。更适合的逻辑是：

> 用户从结果页或我的成果中选择一个已生成作品 → 填标题 → 选择标签 → 是否允许同款 → 是否挂到交换墙 → 发布。

这样社区内容会天然围绕 Remuse 的核心能力，而不是变成泛内容社区。

---

### 5. 我的页：不要做设置页，要做“个人记忆资产总览”

“我的”页面不应该只是头像、设置、退出登录。它应该让用户看到自己在 Remuse 中沉淀了什么。

顶部是用户信息卡：

```text
旧物收藏家
已保存 18 件物品 · 生成 9 个成果 · 开启盲盒 6 天
[开通会员]
```

中间放几张资产入口卡：

| 模块   | 说明               |
| ---- | ---------------- |
| 我的藏品 | 用户上传并保存的物品       |
| 我的故事 | 用户录入的语音故事和文本摘要   |
| 我的成果 | 贴纸、卡片、拼豆图纸、手账页   |
| 我的成就 | 成就图鉴、头像框、风格解锁    |
| 我的交换 | 我发起的交换、别人发来的交换请求 |
| 我的收藏 | 收藏的灵感广场帖子        |

新 PRD 中“我的页面”包括我的藏品、我的故事、我的成就、我的盲盒、我的交换、会员入口和设置，所以这一页应该承担“个人记忆资产中心”的作用。

成就图鉴不要做成传统积分系统，而要偏审美和身份感。比如：

```text
记忆守门人
保存 10 段物品故事

色彩炼金术士
生成 10 张高饱和风格贴纸

转生艺术家
完成 3 次创意再生
```

这也符合 PRD 中“用审美成就而非枯燥打卡激励用户持续使用”的方向。

## 四、视觉风格与组件规范

整体视觉建议使用“深色夜间展厅 + 荧光生命感”的方向。背景可以是接近黑色的深灰、墨绿、深蓝黑，不要大面积纯白。主强调色建议继续使用荧光黄绿色，辅助色使用冷青色，少量暖橙或玫红只用于情绪提示、盲盒奖励和成果完成感。

卡片应该有轻微发光边框，但不要过度赛博朋克。Remuse 的气质不是“冷科技”，而是“有温度的数字博物馆”。PPT 中品牌使命是“用科技守护情感，让生活充满期待”，所以界面不应该太冰冷，要把科技感和情绪感结合起来。

核心组件建议统一如下：

| 组件    | 设计要求                       |
| ----- | -------------------------- |
| 主按钮   | 高亮荧光色，圆角大，文案直接指向下一步        |
| 次按钮   | 深色描边，不抢主按钮                 |
| 物品卡片  | 图片占比高，文字少，保留标签             |
| 故事卡片  | 像档案纸片或记忆摘录，适合显示摘要          |
| 推荐卡片  | 强调“主推荐”和“为什么”              |
| 灵感卡片  | 双列瀑布流，图片优先，文字克制            |
| 状态标签  | 收藏留存 / 创意再生 / 数字化再生 / 交换 等 |
| 麦克风按钮 | 上传故事页核心控件，按住时要有动效          |
| 空态模块  | 温柔引导，不要写“暂无数据”             |

文案一定要避免技术术语。不要出现：

```text
AI 请求失败
JSON 解析错误
Token 已过期
上传接口异常
```

应该改成：

```text
刚刚没有听清，可以再试一次
这张图片暂时没有上传成功
记忆保存失败了，但你的故事还在
图片走丢了，点一下重新加载
```

此前 PRD 也要求用户可见文案统一使用简体中文，避免技术术语、英文占位和无意义提示。

## 五、关键交互状态设计

上传链路必须把状态做完整，因为这是用户第一次感知产品是否可靠的地方。

上传页至少要有这些状态：未上传、上传中、识别中、识别完成、录音中、转文字中、分析中、结果生成中、保存成功、保存失败。不要让用户看到一个无限转圈，也不要在失败后清空用户输入。现有项目文档也强调，这不是纯前端 demo，多数功能依赖服务端持久化、AI 调用、配额和权限，所以前端必须认真处理异步状态。

社区链路至少要有：列表加载、空列表、帖子详情加载、点赞成功、点赞失败、收藏成功、发布中、发布成功、发布失败、图片加载失败。尤其是点赞收藏，可以做乐观更新：用户点了先变亮，接口失败再恢复。

一键同款再生的状态要清晰。用户点击后，不应该直接跳到一个空上传页，而应该在上传页顶部显示：

```text
正在复用「春天感贴纸」风格
上传你的物品，生成同款再生作品
```

这样用户知道自己为什么来到上传页。

会员和付费功能不要过早打断主链路。基础上传、基础识别、基础贴纸 / 卡片生成、今日盲盒、灵感广场浏览应保持免费；高级风格、深度故事整理、年度记忆总结、稀有度深度报告、批量整理可以作为付费点。新 PRD 也明确免费功能和付费功能的边界，付费价值应围绕“更好看、更准、更深、更私密”。

## 六、首版设计稿交付范围

首版不要一次性画几十个页面，容易失焦。建议先画 8 个关键页面，足够支撑产品讲清楚，也方便前端开发。

| 优先级 | 页面         | 说明                   |
| --- | ---------- | -------------------- |
| P0  | 首页         | 包含上传入口、今日盲盒、灵感精选     |
| P0  | 上传故事页      | 包含拍照、识别、语音、文字编辑状态    |
| P0  | 结果页        | 展示故事摘要、情绪回应、推荐方案、下一步 |
| P0  | 灵感广场       | 双列瀑布流，真实社区入口         |
| P0  | 帖子详情页      | 支持查看、点赞、收藏、一键同款、交换   |
| P1  | 发帖页        | 从我的成果发布到灵感广场         |
| P1  | 我的页        | 个人资产总览、成就、交换、会员入口    |
| P1  | 我的藏品 / 成果页 | 展示用户已保存内容和生成结果       |

这套设计稿的重点不是“页面多”，而是把三个核心体验讲清楚：

> 第一，上传一件物品后，系统真的会听故事、分析并给建议。
> 第二，用户不需要自己进入再生工坊判断功能，结果页会直接告诉他下一步。
> 第三，灵感广场是真社区，用户可以看别人、发自己、一键同款、发起交换。

最终小程序前端应该呈现出的感觉是：

> 用户打开它，不是在使用一个工具，而是在进入一个会倾听旧物故事、帮旧物选择第二人生、还能看到别人如何再生旧物的随身数字博物馆。
